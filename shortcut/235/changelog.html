• Added the ability to scan receipts that are then saved to a separate folder in iCloud (Dropbox has too many limitations)
• When you send your completed expenses at the end of the week your receipts will be automatically compressed to a zip file and added to the Email/iMessage
• Expense data is now displayed in a table when sending by email (sends as an html file in iMessage)

5.0 1 year, 9 months ago
iOS 12
What’s new in version 5.0

• Added the ability to start a new week. Doing this automatically archives the data from the previous week into a zip file and adds it to an archive folder adding the date the previous week commenced as part of the file name
• Added an option to send the expenses data and a total via iMessage or Email
• Added an option to request a payment. The weeks total should auto populate the amount you wish to request, however, this is untested as these options (Pay Cash/Venmo etc) aren’t available to the UK yet

NB. When you are satisfied that the iMessage/Emails are sending correctly you can turn off “show when run” and “show compose sheet” on those actions if you want a more seamless experience
