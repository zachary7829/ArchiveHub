1.3.0 1 year ago
iOS 12
Compiled output now uses dictionary values to Passthrough arguments instead of writing to disk between calls. This is a major performance improvement and lays the groundwork for fixing the recursion boundary bugs with data types and files being malformed at the boundary in many use cases which has presently required the adaptation of code to conform to the boundary limits. This change set enables a rewrite of that boundary. The 1.3.x series will focus on performance, code quality, and bug fixing.

1.2.7 1 year ago
iOS 12
Added diff tools. Compile now replaces UUIDs for each dependency with new identifiers to prevent collisions which created unstable build artifacts when two dependencies were made using a duplicate of one. This left identical unique identifiers in the shortcuts corrupting the output. Most compile issues have been resolved with this increment. After some performance improvements the uuid replacement will be applied to the final artifact so that projects can be nested in each other without uuid collision.

1.2.0 1 year ago
iOS 12
Major overhaul of work flow. Projects are managed at the top level based on what is active rather than having to drill down for project management.

The app is now more focused on project build and file management over project and subroutine management. This makes it quicker to write code and test it than before and reduced the steps needed to switch projects. It leaves some less than ideal interfaces behind for project and subroutine interactions but that should improve over time.

Versioning and source control are the next focus which this change set lays the foundation for.
