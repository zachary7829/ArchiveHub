General/Music Library
• FINALLY: Advanced automatic light/darkmode detection

Thanks to u/ROP, Shorty Music now supports the advanced detection.

1.3.3 6 months, 2 weeks ago
iOS 13
What's new in v.1.3.3?

This release has some new features but it focuses more on improvements and bug fixes to make Shorty Music even better :)

-----IT'S REALLY RECOMMENDED TO INSTALL THIS UPDATE-----

[Content updates]

(General)
New: Automatic design detection
• With supermamon's built-in IsDarkMode shortcut, it's possible that Shorty Music can capture the device's appearance. If your device is in the light mode, the design at Shorty Music also gets light, if it's dark, it gets dark.
For now, it can't change the Light-/Darkmode, when you're already in a song profile.
-> Simply go to the settings >> Design >> Automatic

(Explore)
Playlist Centre
• New: Add current song to a playlist
Does that as the description says. You can also add the currently listening song that you haven't added to the music library yet.

(Control Centre)
New: Playback Devices
• You can now change your output devices (speakers, headphones, etc.) at any time
New: Noice Cancellation
• "Noice Cancellation" lets you set the noise cancellation of your headphones

[Improvements & Optimizations]

(General)
• Credits sheet has been added
• Contact sheet has been added
• New menu design for settings >> Design
• A message with an explanation shows up, when an user don't have any music in his Apple music library

(Explore)
• New menu design for playlists charts, discover, mixtape and newcomers

[Bug Fixes]

(Explore)
• The suggestion "You haven't heard that for a long time" couldn't be called
-> This error has been fixed
• The suggestion "A song you don't a lot" couldn't be called
-> This error has been fixed
• "A song you don't hear much" - Any song could be played or assigned to a queue that did not match the suggest.
-> This error has been fixed
• "A song you don't hear much" - When the user goes to "Back", the play menu of this suggestion opens again.
-> This error has been fixed

(Music Library)
• For the most part, no text appears on the music song profile
-> This error has been fixed
• When you wang to open the title search, you get back to the main menu
-> This error has been fixed

I sincerely apologize for the many errors in version 1.3.2.
A lot of things were changed inside the program so that these errors occurred.

[Changes]

(General)
• Leave & Exit has been removed because this feature was superfluous. Press "Cancel" to exit the program.

(Control Centre)
• "Volume" has been removed

What's new in v.1.2.3?

This release focuses on small improvements and bug fixes to make Shorty Music even better :)

[Shorty Music presents:]

Dark Mode

In line with the release of iOS 13, Shorty Music finally supports Dark Mode in the Music Library!

[Improvements]

General
• "Swing Updater" has been replaced by "QuickUpdate". This update has been integrated into Shorty Music. This means no longer a binding separate installation of third shortcuts and you will always be informed directly
• It was cleaned up and the garbage was brought out

Configuration
• Stability optimizations and configuration simplification have been made

[Changes]
• @ROP's I Am Genius has been removed to make Shorty Music free of separate third-party installations
• Connect, Shazam, and other apps have been removed because they cause problems on some devices

[Bug Fixes]

Music Library
• If you want to get information about the current song but you're not listening to music, an "idle document" will appear
-> This error has been fixed
• For different devices, HTML scaling is drastically different.
-> This error has been fixed
